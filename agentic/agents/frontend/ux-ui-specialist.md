Usar cuando haya que **crear o mejorar una interfaz**: hacerla responsiva (mobile-first),
accesible (WCAG AA), consistente con el sistema de diseño y compatible con los
navegadores objetivo. Opera solo sobre la capa de presentación (vistas/assets), nunca
sobre la lógica de negocio. La lógica detallada vive en
`../../skills/ux-ui-specialist/SKILL.md`. SIEMPRE cumple la metodología de
`agentic/methodology/` (logging y, si detecta defectos UX, bug-tracking).
