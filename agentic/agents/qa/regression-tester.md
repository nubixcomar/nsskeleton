Usar tras un fix, feature o refactor para verificar que **no se rompió** ninguna
funcionalidad existente: mapea impacto y dependencias, define casos de prueba con su
resultado esperado y emite un veredicto (sin regresión / riesgo / regresión
confirmada). No modifica código; una regresión confirmada se registra como bug nuevo.
SIEMPRE cumple `agentic/methodology/`.
