## Resumen ejecutivo
La API REST genérica ahora pagina, filtra y ordena, soporta scopes por token (read/write)
y rate-limit por token.

## Cambios realizados
- **`Core\Model::tableName()`** público (para consultas externas seguras).
- **Migraciones**: `scopes` en `api_tokens`, tabla `rate_limits`.
- **`RateLimiter`** (nuevo): ventana fija por clave, fail-open si la base falla.
- **`ApiToken`**: `generate(scopes)`, `normalizeScopes`, `resolve()` (admin + scopes + id).
- **`ApiController::authenticate($scope)`**: valida token, exige scope y aplica rate-limit
  (401/403/429).
- **`ResourceController::index`**: `?page`/`?per_page` (con tope), filtros por campo
  whitelisted (LIKE para texto, igualdad si no), `?q` global, `?sort`/`?order` whitelisted;
  responde `{data, meta}`. Mutaciones requieren scope `write`.
- **Alta de token**: selector de permisos (lectura / lectura+escritura) en el panel.
- **`config/api.php`**: `rate_limit`, `per_page`, `max_per_page`.

## Verificación
- `php -l` OK.
- **Suite**: **196/196 PASS** (+2: `normalizeScopes`, `RateLimiter` límite/bloqueo).
- **E2E (MySQL 3307)**: `GET /api/clientes?per_page=2` → `data`+`meta`; `?nombre=ACME`
  filtra; POST con token **read** → **403**, con **read+write** → **201**; sin token →
  **401**; con el contador al tope → **429**. Tokens/datos de prueba limpiados.
