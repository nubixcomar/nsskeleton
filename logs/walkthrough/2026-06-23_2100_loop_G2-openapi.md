## Resumen ejecutivo
La API ahora se autodocumenta: genera un spec OpenAPI 3.0.3 desde `config/api.php` y ofrece
un visor self-contained.

## Cambios realizados
- **`App\Services\OpenApiGenerator`** (nuevo): arma el spec OpenAPI 3.0.3 (info+versión,
  `servers:/api`, `securitySchemes.bearerAuth`, schemas por recurso, paths CRUD con
  parámetros de paginación/filtros).
- **`Api\DocsController`**: `openapi()` (JSON) + `ui()` (visor HTML self-contained, sin
  dependencias externas → CSP-friendly).
- **Rutas** (antes del wildcard `/api/{resource}`): `/api/openapi`, `/api/openapi.json`,
  `/api/docs`. Link "📖 Documentación API" en el panel de tokens.

## Verificación

- **Suite**: **200/200 PASS** (+4 unit `OpenApi`: versión 3.0.3, bearerAuth, get/post por

- **E2E (MySQL 3307)**: `/api/openapi` → 200 JSON con `/clientes` + `bearerAuth`; `/api/docs`

## Decisiones / fixes
- El server embebido de PHP (`php -S`) intercepta paths con `.json` y no enruta → se agregó
  `/api/openapi` (sin extensión) que anda en dev y en Apache; `.json` queda para Apache/tools.
- Visor propio (sin Swagger UI por CDN) para respetar la CSP local; el `openapi.json` es
  importable en Swagger UI/Postman externos.
