## Resumen ejecutivo
Se agregó una capa API REST con autenticación por Bearer token y CRUD genérico sobre
los modelos registrados, más la gestión de tokens en el panel. Verificada end-to-end.

## Cambios realizados
- **Migración**: `20260623_0009_create_api_tokens.sql` (hash del token; con `-- @DOWN`).
- **`App\Services\ApiToken`**: `generate` (devuelve el token en claro una vez),
  `validate` (admin activo + actualiza last_used), `all`, `revoke`.
- **`Api\ApiController`**: autenticación Bearer (header `Authorization`, fallback
  `$_SERVER`) + respuestas/errores JSON.
- **`Api\ResourceController`**: CRUD genérico (`index/show/store/update/destroy`) sobre
  recursos de `config/api.php` (whitelist de campos, casteo de bool).
- **`config/api.php`**: registro de recursos (por defecto expone `productos` si existe).
- **Panel**: `Admin\ApiTokenController` + vista `admin/api/index` (crear/revocar) + menú.
- **Rutas**: `/api/{resource}` (GET/POST) y `/api/{resource}/{id}` (GET/PUT/PATCH/DELETE)
  + `/admin/api-tokens`.

## Verificación

## Decisiones de diseño
- Token con prefijo `nsk_`, solo hash persistido; el valor se muestra una vez.
- CRUD genérico por configuración (`config/api.php`) → no hay que escribir un controlador
  por recurso; el generador de módulos podrá registrar recursos a futuro.
- Auditoría: creación/revocación de tokens quedan registradas (`api_token.*`).
