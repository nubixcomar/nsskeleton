## Resumen ejecutivo
Se agregó modo oscuro al backend con toggle persistente, sin necesidad de recompilar
Tailwind (vía una hoja de overrides bajo `.dark`).

## Cambios realizados
- **`system/public/assets/css/theme.css`**: reescribe los colores de las utilidades más
  usadas (bg-white, slate-*, ring/border/divide, inputs, badges) cuando `<html>` tiene
  la clase `dark`.
- **`Core\Assets::head`**: enlaza `theme.css` (en ambos modos) + un script inline que
  aplica el tema guardado **antes del primer pintado** (anti-flash).
- **Toggle 🌓** en el header del backend: alterna `.dark` y persiste en `localStorage`.

## Verificación
- `php -l` OK.
- **Suite**: `php tests/run.php` → **146/146 PASS** (+4 unit `Theme`: existe theme.css,
  define overrides `.dark`, `head()` incluye el link + el init de localStorage).
- **Smoke (servidor)**: `theme.css` → 200 `text/css`; `/admin/login` incluye el link y el
  init `classList.add("dark")`; el toggle aparece en el backend.

## Decisiones de diseño
- Overrides CSS bajo `.dark` en vez de `dark:` de Tailwind → no hay que recompilar el CSS
  ni editar decenas de vistas; el binario de Tailwind no es necesario para esta fase.
- Preferencia en `localStorage` (cliente) + init anti-flash en `<head>`.
- Funciona igual en modo `local` y `cdn`.
