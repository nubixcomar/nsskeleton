# app-agentic/knowledge/ — Conocimiento del proyecto

Manuales y referencias de **dominio de tu sistema** (APIs propias, reglas de negocio,
integraciones específicas). Mismo criterio que el knowledge del core
([`../../agentic/knowledge/README.md`](../../agentic/knowledge/README.md)): un archivo por
dominio/plataforma, versionado en su encabezado, referenciado por tus skills (no duplicado).

Ejemplos de qué va acá:
- Manual de una API propia del sistema.
- Especificación de un dominio de negocio (facturación, WMS, etc.).
- Integraciones puntuales que no son parte del core.
